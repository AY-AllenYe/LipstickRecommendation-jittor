本项目旨在根据用户上传的口红色号提供几款相似（主要是颜色方面）的口红。

## version 1.x

### 说明

**1.数据集**

​	我在 Github 上进行了搜索到了 [lipstick.json](./datasets/lipstick.json)，该 JSON 文件其中包含了 288 种不同口红的属性信息。该 JSON 文件展示了每款口红的基本属性，比如品牌、系列、名称、编号，以及最为重要的部分——十六进制颜色。

​	我编写了 Python 脚本来处理十六进制颜色，将十六进制颜色转换为 RGB、HSV 格式，然后创建了一个 CSV 文件来存储 JSON 文件所显示的信息以及我处理后的属性。此外，我还根据十六进制颜色的值生成了 288 张纯色的 JPG 图片。

​	我将数据集进行了划分：95%用于训练（272 个样本），5%用于验证（15 个样本），且未进行随机排列。

**2.分类方法**

​	**K-Means**是主要分类方法。

​	为了决定分类的类别数量（K-Means 中的 “K”），我编写了 for 循环注意计算与尝试 WCSS 和轮廓得分来决定最佳K值。

​	在该程序中，我还通过不断增加 K 值的方式绘制了 WCSS 曲线和轮廓得分的图形，并自动选择了最佳的 K 值。确定后，程序会展示数据的分布情况和聚类结果。此外，程序自身还创建了两个新的 CSV 文件，一个用于保存聚类名称和 ID，另一个用于存储聚类 ID 与相关聚类名称的字典。

​	本机得出最佳值为 K=5.

**3.训练**

​	**ResNet50** 作为训练网络（本项目基于jittor框架开发。未来会尝试开发tensorflow或者pytorch版本）。

​	根据训练集样本数量（272 个样本），我设置 batch_size 为 68，每个epoch跑4次。同时根据 K-means 的最佳 K 值设置 num_classes 为 5. 共计训练 80 轮（epoch = 80）。

**4.结果**

​	80 轮的训练花费了约 1 个小时，训练的模型大小约为 9MB.

​	最佳训练模型（训练至第 80 轮得到）在训练集的命中率为 98.53%。经过测试，在 15 个样本的测试集上正确分类了 14 个。

**5.推理**

​	支持单一样本处理或者多样本批量处理。

​	要对单个样本进行分类，用户需要在命令中输入十六进制颜色代码。

​	若要对批量样本进行分类，用户需要一个文件夹来存放 JPG 标准色图像（PNG 图像也可以，但需要在 [infer.py](infer.py) 中修改一行代码）。

```
文件目录树结构
|-- main
    |-- App.py                          // 界面设计.
    |-- dataset_proceed.py              // 处理 JSON 并生成 CSV/JPG.
    |-- infer.py                        // 使用模型推理预测.
    |-- train.py                        // 使用 CSV/JPG 训练模型.
    |-- videocapture.py                 // 视频实时监控模块.
    |-- README.md
    |-- datasets
    |   |-- lipstick.json               // 原始数据集 JSON.
    |-- utils
    |   |-- cluster.py                  // 计算 K-Means 最佳 K 值与分类.
    |   |-- compute_weights.py          // 设计训练权重.
    |   |-- csv2dict.py
    |   |-- hex2hsv.py
    |   |-- hex2rgb.py
    |   |-- json2csv.py
    |   |-- load_dataset.py             // 加载数据集.
    |   |-- logger.py                   // 自动记录与打印日志.
    |   |-- models.py                   // ResNet50.
    |   |-- recommend.py                // 生成推荐代码.
    |   |-- rgb2hex.py                   
```

```
python train.py

```

### 依赖

​	本机（Windows）使用以下代码库与相应版本，能够成功测试。其中，旧版本的 jittor （最新版本的 jittor 暂未支持 Windows 系统）需要指定的 numpy 库匹配（否则会报错）。因此，强烈建议安装以下版本（尤其是 numpy 和 jittor）。其他库文件默认安装即可。

```
jittor == 1.3.8.5
numpy == 1.26.4
PIL == 12.0.0
pandas == 2.3.3 
tqdm == 4.67.1
sklearn == 1.7.2
dlib == 20.0.0
以及其他。
```
